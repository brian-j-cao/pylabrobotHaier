
# 海尔台式全自动液体处理工作站 设计规格文档
第一阶段（第一阶段）设计主要是针对血样生物样本库预处理的功能实现主要覆盖白膜层、血浆分装及 PBMC全自动分离分装的场景

## 一、基本功能设计规格表（第一阶段，HEPA 功能不实现）

| 项目 | 设计规格 / 要求 | 备注 |
|------|-----------------|------|
| 产品定位 | 台式全自动液体处理工作站，面向离心后血样的生物样本库预处理及PBMC全自动分离分装 场景| 第一阶段 聚焦血浆/PBMC 等前处理 |
| 目标应用 | 离心血样分装、冻存管分装、管架/板重排、样本质量初筛（分层 QC） | 后续可扩展通用 protocol |
| 源样本容量（分血） | 一次上架处理 **≥96 支** 已离心、已开盖的采血管 |  |
| PBMC全自动处理流程 | 参考武汉病毒所方案，可对接样本管开盖及离心设备，可对接自动开盖机 |  |
|拓展对接功能| 可对接样本管开盖机、离心机、堆栈等设备 |后期支持细胞计数及酶标仪等设备  |
| 台面SBS板位容量 | 支持 ≥1 个 SBS 冻存管架（8×12），或 96 孔板 1–2 块 ||
| 源管条码识别 | 支持 1D 条码；首读率 ≥ 99.5% | 工业读码器 + 多角度重试 |
| 冻存管 2D 识别 | 支持 SBS 冻存管 DataMatrix 2D 底码；解码率 ≥ 99.9%，且只占用一个板位| |
| QC 相机配置 | 背光成像舱 + 固定焦距工业相机 + 遮光结构 | 覆盖整管高度范围 |
| QC 识别内容 | 识别血浆/白膜层/红细胞界面；提取界面高度、清晰度、溶血代理（颜色）、液面高度等 |  |
| QC 精度指标 | 分层界面分类准确率 ≥ 98%；界面高度 RMSE ≤ 1.0 mm（对标人工标注） | 以标注数据集验证 |
| 去盖器对接能力 | **第一阶段 必须支持** SBS 冻存管去盖器集成；右侧预留标准对接口平面；夹爪可夹取冻存管盒 | 统一高度与参考面 |
| 去盖流程 | 支持整架去盖 → 2D 底码读取 → 上架至甲板目标位的自动流程 | 去盖失败需告警和处理引导 |
| 去盖可靠性 | 去盖成功率 ≥ 99.7%（1000 次测试）；可检测卡盖/异常力矩 | 需监测扭矩/位置异常 |
| HEPA 生物安全（第一阶段） | **第一阶段 不实现 HEPA 模块及其功能**；仅在机械/电气设计上考虑未来可加装的可能性，不对风速/颗粒/联锁性能做指标承诺 | 具体 HEPA 性能在后续 Phase 定义 |
| 移液通道数 |灵活 4 或 8 通道移液头 | 第一里程碑 1 不做 96/384 通道 |
| 移液tip支持范围 | 典型 10–1000 μL（具体根据选型补充） | 满足血浆/细胞组分分装 |
| 吸头要求 | SBS 规格吸头盒，一次性吸头，系统自动计数与缺失检测 | 至少支持 96 个以上吸头容量 |
| 移液精度 |（1–1000 μL 范围内） | |

## 二、机械 / 结构设计规格表（第一阶段，仅做 HEPA 结构预留）


| 项目 | 设计规格 / 要求 | 备注 |

| 甲板下平面留白 | 甲板下方平面投影面积中，**≥60% 区域无电子元件/结构阻挡** | 用于后续下置模块整合 |

| 线缆与风路 | 电源/控制等集中在底部后部；两侧和后侧的对接区域不走线路与电控件 | 便于模块集成与散热 |
| 未来 HEPA 预留 | 顶部结构需预留一定平面与承载能力，以便未来安装 HEPA 模块（风道/风速细节暂不设计）；不在 第一阶段 内实现气流功能 | 仅结构/强度预留 |

| 工业设计风格 | 现代、简洁、“洁净实验室”视觉；正面切线最少化；无外露螺钉 | 与新一代自动化样本库产品家族风格统一 |

|自动样本管上样系统| 可选配自动样本管上样系统，单次最大上样数量不少于96管，可适配多种样本管；方便拆卸和恢复为通用版位；具备自动上样扫码和分层识别功能 | 预留电气接口 |

## 三、电气 / 控制设计规格表（第一阶段，预留 CAN 备用接口，不实现 HEPA 功能）


| 项目 | 设计规格 / 要求 | 备注 |
|------|-----------------|------|
| 供电输入 | AC 100–240 V，50/60 Hz，单相，整机功率目标 ≤ 800 W | 第一阶段 不包含 HEPA 模块功耗 |
| 内部电源架构 | 统一 **24 V DC 主电源母线**，通过 DC-DC 转换生成 **5 V / 3.3 V** 逻辑电源；24 V 供电用于电机、阀、继电器等功率负载 | 2 |
| 电源分配 | 使用 **电源分配板（PDB）**：AC 入口 → EMI 滤波 → 主开关 → 24 V 电源 → 分路保险/断路器 → 运动控制、IO、外设（如去盖器） | 分路清晰、便于维护、功率计算需考虑后续模块的拓展性 |
| 接地方案 | 机壳保护接地（PE）与 24 V 0V 采用星形接地方式；所有外露金属件等电位连接 | 符合相关电气标准要求 |
| 控制拓扑 | “**上位机 PC + 主控板 MCU + 现场总线节点**”架构：PC 负责 UI/业务逻辑；主控板负责运动与 IO 实时控制；现场设备通过 CAN / RS-485 接入 | 便于扩展与升级 |
| 上位机 | 工业 PC 或嵌入式 x86/ARM 计算单元，运行 Windows ；通过以太网或 USB 与主控板通信 | 支持本地离线运行 |
| 主控板/主控PLC | MCU实现运动的协调、IO 扫描、安全监控；提供 Ethernet/USB 与 PC 通讯；集成 CAN、RS-485 接口 |需考虑制冷、震荡模块的拓展性  |
|移液臂控制器| 独立运动控制器，负责移液臂的多轴运动控制与位置反馈，通过本地的CAN总线与各移液器和XY轴电机进行通讯；通过 CAN 总线与主控板通信 | 支持多轴协调运动 |
|移液通道| 独立控制单元，负责移液头的吸液/分液动作控制，通过本地的CAN总线与移液臂控制器进行通讯| 第一阶段计划选用外购带独立z轴的控制模块|
|夹爪臂控制器| 独立运动控制器，负责夹爪臂的多轴运动控制与位置反馈，通过本地的CAN总线与夹爪电机进行通讯；通过 CAN 总线与主控板通信 | 支持夹取冻存管盒动作 |
| CAN 总线（运动） | 所有轴上运动控制模块、autoload 等拓展模块通过 CAN 总线与主控板通讯；使用CANopen 类协议；每个模块、电机驱动为独立节点 | 线束简化、扩展性好 |
| **CAN 预留接口** | 主控板需 **至少预留 2 路未使用的 CAN 物理连接器**（带终端可配置），电气特性与主 CAN 总线一致，用于未来外部模块扩展 | 第一阶段不使用，但硬件必须实现 |
| IO 架构 | 主控板集成基础 DI/DO/AI，必要时通过 CAN/RS-485 外扩 IO 模块；门锁、限位、泄漏、风扇等传感器接入 IO | 为未来增加传感器保留余量 |
|对外供电接口| 预留对外供电接口（AC + 24V DC），用于对接外部模块如开盖机、堆栈等设备 | |
| 外设总线接口| 推荐使用 **RS-485（Modbus RTU）** 或以太网（Modbus/TCP/REST）与去盖器通讯；预留 1 路 RS-485 + 1 路以太网给去盖器 | 去盖器纳入急停与安全联锁 |
| HEPA 控制 | 第一阶段 **不实现 HEPA 功能，不与任何 HEPA 模块电连接**；主控板可仅在原理图级预留模拟量/开关量引脚+CAN接口，不必装配实际接口 | 后续再定义具体接口 |
| 急停回路 | 硬件急停回路（安全继电器 + 常闭触点链）：串联前面板急停、维护门联锁；急停动作 ≤100 ms 切断所有运动及危险电源 | 主控板同步采样急停状态用于日志 |
| 传感器接口 | 配备： 内部温湿度传感器、各到位传感器；  | 支持基本安全与维护 |
| 状态指示光带 | 状态光带由主控板 PWM/恒流驱动，支持多色控制；状态机集中在主控软件，由 PC/MCU 统一驱动 | 亮度可调 |
| I/O 面板与连接 | 电源插座、以太网、USB、RS-485/CAN 等外部接口集中在后部 I/O 面板；采用带锁紧结构的工业连接器或 RJ45; 内部针对自动上样模块、扫码模块、震荡模块、制冷模块等 | 面向用户的线缆布局简洁 |
|Deckview 监控相机| 甲板顶部或侧面布置至少 1 台监控相机，用于实时监控板面操作状态及故障诊断，后续搭配ai自动纠错功能| 分辨率至少 5 MP |
| EMC 设计 | 按企标执行 | 降低相机/读码器干扰 |
| 安全与合规 | 电子设计需满足 CE/UL 等目标市场标准（爬电距离、绝缘、过温/过流保护等）；关键器件需具备合规认证 | 具体标准由注册策略细化 |

## 四、软件 / 功能设计规格表（第一阶段，暂不实现 HEPA 逻辑）


| 项目 | 设计规格 / 要求 | 备注 |
|------|-----------------|------|
| 运行模式 | 本地 PC 客户端控制，离线运行；支持保存/加载 protocol；支持批量 CSV 导入样本清单 | 第一阶段 不提供 SDK/API |
| 功能定位 | 支持生物样本库常用流程：管→冻存管分装、管→板分装、管架/板重排、QC 闸门控制 | 之后可扩展其他 protocol |
| QC 流程逻辑 | 上架后自动完成：条码读取 → 相机分层识别 → QC 通过/不通过判定；仅通过样本进入移液队列 | 支持人工带理由放行 |
| 去盖逻辑 | 对冻存管流程强制执行“去盖 → 2D 底码读取 → 目标位确认”后才允许移液 | 任一步失败必须有错误提示 |
| HEPA 逻辑（第一阶段） | **第一阶段 软件不实现 HEPA 风量监测、联锁和告警逻辑**；架构允许未来通过配置/模块加载增加 HEPA 状态和安全联锁 | 后续 Phase 再引入 |
| 审计与数据 | 自动生成运行日志与映射清单：源管 ID、管架位次、QC 指标/状态、目标孔/2D、分装体积、时间戳、操作员、外设事件（如去盖器） | 默认支持 CSV/PDF 导出 |
| 错误处理 | 条码/2D 读取失败、QC 置信度低、去盖异常等需有清晰错误代码和提示文案，并支持局部重试/跳过 | 避免整批作废 |
| 用户管理 | 支持基础用户/管理员角色；重要操作（人工放行、急停解除等）需绑定用户身份 | 日后可扩展审计合规 |
| 界面语言 | 初版至少支持中文界面；界面用词简洁、步骤清晰，错误提示聚焦“问题 + 操作建议” | 便于一线技师理解 |
| 日志留存策略 | 默认本地保存运行日志与 QC 结果，具备导出/备份能力；留存时长可配置（例如 1–3 年） | 图像留存策略可在后续版本细化 |
